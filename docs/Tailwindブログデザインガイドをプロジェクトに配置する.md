あなたは Ruby on Rails + Tailwind CSS 4.1 を用いた
ブログ / メディアサイトのUI設計・運用に精通したテックリードです。

Bootstrap 5 から Tailwind CSS 4.1 へ段階移行中のブログプロジェクトに対して、
「Tailwindブログデザインガイド」をプロジェクト内に正式配置し、
今後の実装・レビュー・拡張の“唯一の基準”として機能する文書を作成してください。

# 目的（最重要）
- ブログとして「長文が読みやすい」UIを一貫して維持する
- 実装者が余白・文字サイズ・行間で迷わないようにする
- Tailwindクラスの乱用・デザインブレを防ぐ
- Bootstrap時代の感覚を完全に断ち切る

# 前提
- フレームワーク：Ruby on Rails
- CSS：Tailwind CSS 4.1
- サイト種別：ブログ / 技術メディア
- 記事本文は HTML / Markdown の両方が存在し得る
- Tailwind は Bootstrap と共存中（ページ単位で移行）
- 管理画面ではなく「読み物」が主役

# ガイドに必ず含める章立て

## 1. このガイドの位置づけ（Must）
- 何のためのガイドか
- いつ参照するか（新規画面/既存改修/レビュー時）
- Tailwind公式Docsとの役割分担

## 2. デザイン思想（Why）
- ブログUIで最優先すべきは「可読性」であること
- 余白・行間・文字サイズを数値で決める理由
- Bootstrapデフォルトとの思想的な違い

## 3. レイアウト基準（Must）
- 記事本文の最大幅（max-w-* 推奨値）
- 一覧ページの最大幅
- コンテンツ中央寄せの原則
- ブレークポイント別の左右padding指針
  - mobile / tablet / desktop

## 4. 文字サイズ・行間ルール（最重要・Must）
- body 基本文字サイズの基準
- 行間（leading-*）の基準値
- 見出し（h1〜h4）の
  - text-*
  - leading-*
  - margin（上下余白）
- モバイルで文字を小さくしすぎないルール

## 5. 記事本文（prose）の設計指針（Must）
- Tailwind Typography（prose）を使う場合の推奨設定
- prose を使わない場合の代替ルール
- 段落・リスト・引用・コードブロックの余白基準
- 1行あたりの文字数の目安（可読性）

## 6. 記事一覧・カードUIの余白ルール（Should）
- カード内 padding の基準
- カード間 gap の基準
- タイトル・メタ情報・抜粋の間隔
- hover時は「動かしすぎない」理由

## 7. 色・コントラストの基本ルール（Should）
- 基本文字色（slate系など）
- 背景色の使い分け
- アクセントカラーは1色までに抑える理由
- NG例（コントラスト不足、派手すぎ）

## 8. レスポンシブ設計の考え方（Must）
- モバイルファーストの余白設計
- ブレークポイントごとに
  「変えてよいもの / 変えないもの」
- 文字サイズをレスポンシブで変えすぎない理由

## 9. Tailwindクラス早見表（実務向け・Must）
- よく使う組み合わせを用途別に表でまとめる
  例：
  - 記事本文
  - 記事タイトル
  - 抜粋
  - メタ情報（日付・カテゴリ）
- NGパターン集（詰めすぎ / 大きすぎ / 行間不足）

## 10. 実装例（Railsビュー・コピペ可）
- 記事本文（article）
- 記事一覧カード
- サイドバー見出し
※ Tailwind CSS 4.1 のクラスで、完全なHTML断片として示す

## 11. 運用ルール（Must）
- 新しいUIを作るときのチェックリスト
- 迷ったらどの章を見るか
- Bootstrap時代のクラス・感覚を持ち込まない注意点
- PRレビュー時の確認観点

# 表現・品質要件
- Markdown形式
- 見出しと箇条書きを多用し、読みやすく
- 抽象論だけで終わらず「数値・クラス名」を明示
- Tailwind CSS 4.1 のクラス名を使用
- 日本語・実務者向け・落ち着いた文体
