あなたは Ruby on Rails（Turbo/Hotwire）と Tailwind CSS 4.1 に精通した
ブログ/メディアサイト向けUI設計の専門家です。

Bootstrap 5 を使用しているブログサイトの
「ヘッダー / ナビゲーション」を、
Tailwind CSS 4.1 でブログ向けに最適化してください。

# ゴール（最重要）
- ヘッダー/ナビを Tailwind CSS 4.1 のみで実装する（Bootstrapクラス0件）
- 読み物（記事本文）の邪魔をしない “静かな” ヘッダーにする
- モバイルでも使いやすい（ハンバーガー/ドロワー）
- Turbo で破綻しない（リンク遷移、戻る、検索）
- 既存サイト全体では Bootstrap を残してよいが、
  Tailwind禁止ゾーン（tailwindレイアウト配下）ではこのヘッダーを使う

# 前提（Rails）
- ERBビュー
- Tailwind専用レイアウト（例：app/views/layouts/tailwind.html.erb）が存在する想定
- ナビは partial 化し、Tailwind禁止ゾーンで共通利用できるようにする
- ルーティングは既存のものを優先し、不明な場合はフォールバック（仮）を用意してコメントする

# 対象（新規/更新してよいファイル）
- app/views/shared/_tw_header.html.erb（新規）
- app/views/shared/_tw_mobile_nav.html.erb（新規・必要なら）
- app/views/layouts/tailwind.html.erb（ヘッダー partial を読み込むために最小変更）
- app/javascript/controllers/mobile_nav_controller.js（Stimulusで実装する場合。任意）
  ※JSを使わない実装（details/summary）案も併記してよい

# UI要件（ブログ向け）
## 1) ヘッダーの構成
- 左：サイトロゴ/サイト名（トップへリンク）
- 右：
  - 検索（任意。qパラメータのGET）
  - ログイン 新規登録
  - ログインしている場合のみ、マイメニューを表示。マイメニューの下には
  　- アカウント詳細
  　を表示するに
- ヘッダーは常時表示（sticky）は任意。入れる場合は控えめな影/境界線。

## 2) モバイルナビ
- ハンバーガーボタン
- 開くとドロワー（画面右or下）
- 登録一覧、お問い合わせ、ログイン、新規登録
- a11y：aria-expanded, aria-controls, focus trap は理想だが最小でもOK
- Escで閉じる（可能なら）

## 3) デザイン方針
- “静か” で余白がある（読み物の邪魔をしない）
- 背景は白〜薄いslate、境界線は薄く
- 文字は小さすぎない（text-sm〜base）
- hover/focus-visible を入れる
- ダークモードは任意（入れるなら自然に）

# Tailwind実装ルール（厳守）
- Tailwind CSS 4.1 クラスのみ
- Bootstrapクラスは一切使わない
- クラスの長文化を避けるため、ヘッダーは partial にまとめる
- アクセントカラーは1色まで（リンクのhoverなど）
- 余白・高さはブログとして自然（詰めすぎない）

# ルーティング/データのフォールバック（必須）
- ルートが不明な場合、以下の候補を tries して存在すれば使う：
  - root_path
  - posts_path / articles_path
  - categories_path
  - tags_path
  - about_path
- 主要カテゴリがDBにある前提なら @categories 等を受け取る設計にしてよいが、
  無ければ固定リンクでもOK（ただし「未接続」コメントを入れる）

# 出力形式（必須）
1) 変更/新規作成するファイル一覧（パス付き）
2) 各ファイルの完成コード全文（コピペで動く）
3) Stimulusを使う場合：必要なコントローラコード全文
4) 動作確認チェックリスト（10項目前後）
5) Bootstrap禁止ゾーン違反がない確認方法（grep例）

# 追加品質要件
- h1 をヘッダーに置かない（記事ページのh1と競合しない）
- ナビはキーボード操作でも使える
- Turboでの遷移が自然（data-turbo="false" は乱用しない）
