あなたは ブログ / メディアサイト向けのUI・タイポグラフィ設計に精通したデザインエンジニアです。
Bootstrap 5 から Tailwind CSS 4.1 に移行するブログサイトのために、
「ブログ用 Tailwind デザインガイド（余白・文字サイズ・行間）」を作成してください。

# 目的
- ブログとして「長文が読みやすい」ことを最優先する
- 管理画面っぽさを排除し、軽く・静かなUIを実現する
- Tailwind CSS 4.1 を前提に、実装者が迷わず使える“基準”を作る

# 前提
- 対象：記事一覧 / 記事詳細 / 固定ページ / サイドバー
- フレームワーク：Ruby on Rails
- CSS：Tailwind CSS 4.1
- Bootstrap は段階的に廃止予定
- 記事本文は HTML / Markdown どちらもあり得る

# 文書の出力先
- docs/frontend/tailwind_blog_design_guide.md

# ガイドに必ず含める章立て

## 1. デザイン思想（Why）
- なぜ「余白」と「文字サイズ」を最優先するのか
- Webメディアにおける可読性の原則（簡潔に）
- Bootstrapデフォルトとの思想の違い

## 2. ベースレイアウト基準
- 最大幅（max-width）の推奨値
  - 記事本文
  - 一覧ページ
- コンテンツ中央寄せの基本ルール
- 左右余白（padding）のブレークポイント別指針

## 3. 文字サイズ・行間ルール（最重要）
- body 基本文字サイズ（text-base / text-lg 等）
- 行間（leading-*）の基準
- 見出し（h1〜h4）のサイズ・行間・余白
- モバイルとPCでのサイズ差の考え方

## 4. 記事本文（prose）の設計指針
- Tailwind Typography（prose）を使う場合の推奨設定
- prose を使わない場合の代替指針
- 段落・リスト・引用・コードブロックの余白基準
- 行の長さ（1行あたりの文字数）の目安

## 5. 記事一覧・カードUIの余白ルール
- カード内余白（padding）
- カード間余白（gap）
- タイトル・メタ情報・抜粋の間隔
- hover時の視覚変化は「控えめ」にする理由

## 6. 色とコントラスト（簡潔）
- 基本文字色の推奨（slate系など）
- 背景色の使い分け
- アクセントカラーは1色に抑える理由

## 7. レスポンシブ設計の考え方
- モバイルファーストの余白設計
- ブレークポイントごとに「何を変え、何を変えないか」
- 文字サイズをむやみに小さくしないルール

## 8. Tailwindクラス早見表（実務向け）
- よく使う文字サイズ・行間・余白の組み合わせ表
  例：
  - 記事本文：text-base leading-7
  - 記事タイトル：text-2xl font-semibold leading-tight
  - 抜粋：text-sm text-slate-600 leading-relaxed
- NGパターン（詰めすぎ / 大きすぎ）

## 9. 実装例（Railsビュー）
- 記事本文（articleタグ）
- 記事一覧カード
- サイドバー見出し
※ 完全なHTML断片として提示する（コピペ可）

## 10. デザインガイドの運用ルール
- 「迷ったらどこを見るか」
- 新しいUIを作るときのチェックリスト
- Bootstrap時代のクラスを持ち込まないための注意点
